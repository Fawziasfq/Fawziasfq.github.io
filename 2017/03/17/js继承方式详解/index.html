js继承方式详解

2017-03-17


js继承的概念

常用方式

  • 原型链继承（对象间的继承）
  • 类式继承（构造函数间的继承）

OO语言支持两种继承方式：接口继承和实现继承，然而接口继承需要签名，而js中没有，所以只能支持实现继承，主要是依靠原型链（prototype机制），或者apply和call方法实现。

如何实现自定义对象：

  • 可以把prototype看作是一个模板，新创建的自定义对象都是这个模板的一个拷贝
  • js可以通过构造函数和原型的方式模拟实现类的功能
  • js类式继承的实现也是依靠原型链来实现的

原型链继承

缺点：字面量重写会中断关系，使用引用类型的原型，并且子类型无法给超类型传递参数

思想：利用原型让一个引用类型继承另一个引用类型的属性和方法（借助已有的对象创建心得对象，将子类的原型指向父类，就相当于加入了父类这条原型链）。

构造函数、原型和实例的关系：每个构造函数都包含一个原型对象，原型对象都包含一个指向构造函数的指针，实例对象都包含一个指向原型对象的内部指针。

确定原型和实例的关系

两种方法：

  • instanceof
  • isPrototypeof()

只要是原型链中出现过的原型都可以说是原型链派生的实例的原型，因此都会返回true

在js中，被继承的函数称为超类（父类，基类），继承的函数被称为子类型（子类，派生类）

存在问题：

  • 字面量重写原型会中断关系，使用引用类型的原型
  • 子类型还无法给超类型传递参数
  • 伪类解决了引用共享和超类型无法传参的问题——借用构造函数

借用构造函数（类式继承）

借用构造函数解决了刚才的两种问题，但是没有原型，复用无从谈起。所以我们需要原型链+构造函数的模式，这个模式称为组合继承

思想：在子类型构造函数的内部调用超类型的构造函数，一般都是组合使用

问题：没有原型，复用无从谈起，因此需要原型链+借用构造函数的模式，称之为组合继承。

组合式继承

组合式继承是比较常用的一种继承方法，其背后的思路是，使用原型链实现对原型属性和方法的继承，而通过对借用构造函数来实现对实例属性的继承。这样，既通过在原型上定义方法实现了函数复用，又保证了让每个实例都有它自己的属性

组合式继承是一种比较常用的继承方法，通过原型链继承原型的属性和方法，通过借用构造函数实现对实例属性的继承。

这样既能在原型上定义方法实现函数复用，又能保证每个实例都有自己的属性。

call()的用法：调用一个对象的一个方法，以另一个对象替换当前对象

原型式继承

这种继承借助原型并基于已有对象创建新对象，同时还不用创建自定义类型的方式称为原型式继承

原型式继承首先在obj()函数内部黄建一个临时性的构造函数，然后将传入的对象作为这个构造函数的原型，然后返回这个临时类型的一个新实例

寄生式继承

这种继承方式是原型式+工厂模式，目的是为了封装创建的过程

组合式继承的小问题

组合式继承是js最常用的继承模式，但组合继承的超类型在使用过程中会被调用两次：依次是创建子类型的时候，另一次是在子类型构造函数内部。
